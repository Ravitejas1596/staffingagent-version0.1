Invoice Matching Agent

Line-level tolerance release on invoices that match cleanly. Rate variances, missing OT lines, and PO discrepancies escalate to the HITL queue with full context. Every automated release is reversible for 7 days — with a permanent audit trail.

Invoice Matching alerts flow into the same Alert Queue as every other agent — with the 7-day reversal window in the footer.
The Problem

Most invoices match cleanly. The ones that don’t are where the money lives.

The majority of invoices coming out of Bullhorn against approved timesheets and active POs match within pennies. Releasing those by hand is pure toil. The residual — rate variances, missing overtime lines, PO amount or expiry issues — is where billing errors and disputes actually happen, and those are the ones you want a human looking at with full context, not buried under the clean ones.

The Decision

Three lanes. Tolerance release, human review, or block.

For every invoice, the agent performs line-level matching against timesheets, active placements, and the governing PO. The output is one of three decisions — each routed appropriately, each logged, each reversible inside the 7-day window.

Release

Clean match — inside tolerance

Every line matches an approved timesheet line, rate equals the active placement rate, total is within the PO’s remaining balance. Released automatically. Reversible for 7 days.

Review

Variance — HITL queue

Rate variance, missing OT line, misaligned pay period, or PO discrepancy. Escalates to the Alert Queue with full context: which line, what’s off, by how much, and what the agent thinks the correction is.

Block

Material failure — block invoice

PO expired, PO balance insufficient, placement closed before the pay period, or duplicate invoice detected. Invoice is held and the alert fires with “CRITICAL” severity.

The Checks

Every invoice gets the same line-level scrutiny as your best billing analyst.

The Invoice Matching Agent runs the same set of checks on every invoice. No invoice is ever released without passing all of them. No invoice is blocked without a named reason. Everything writes to the audit log.

✓

Line-to-timesheet match

Every invoice line reconciled to an approved timesheet line by date, hours, and category.

✓

Rate alignment

Bill rate equals the active placement bill rate for the period, with OT multiplier applied where applicable.

✓

OT presence

Any timesheet line flagged OT produces a corresponding OT invoice line. Missing OT lines escalate.

✓

PO coverage

PO is active, not expired, and has remaining balance ≥ invoice total.

✓

Duplicate detection

Hash over client + period + placement + amount blocks accidental re-bills.

✓

Tolerance window

Per-tenant tolerance in $ and %; anything outside the window lands in HITL, never auto-releases.
