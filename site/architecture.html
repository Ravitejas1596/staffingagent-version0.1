Multi-Tenancy

Enterprise-Grade Tenant Isolation

Shared database with PostgreSQL Row-Level Security. Every query is automatically scoped to the authenticated tenant. Zero cross-tenant data leakage by design.

Step 1

JWT Authentication

Every request carries a signed JWT with tenant_id, user_id, and role claims. HS256 with rotatable secret.

Step 2

Session Scoping

Database session runs SET LOCAL app.tenant_id before any query. Transaction-scoped, auto-cleared.

Step 3

Row-Level Security

RLS policy on every tenant table: USING (tenant_id = current_setting('app.tenant_id')::uuid)

Step 4

Audit Trail

Every agent action and data modification is logged with SHA-256 input hash, timestamps, and approval status.

Assess & Transform Tiers

Shared database, RLS isolation. Cost-effective for $5K–$12.5K/month customers. Same security guarantees via PostgreSQL enforcement.

Enterprise Tier

Option for dedicated database instance per customer. Physical isolation for firms requiring it. Same API, same agent layer — just a different connection string.

API Surface

RESTful API — v1

All endpoints are JWT-authenticated (except health and login). Responses are JSON. Tenant isolation is enforced at the database layer via RLS.

GET /health
Health check. Returns {"status":"ok","version":"0.1.0"}
POST /api/v1/auth/login
Authenticate with email, password, and tenant slug. Returns JWT access token + user profile.
GET /api/v1/dashboard/metrics JWT
Aggregate counts: placements, timesheets, invoices, VMS records, pending review queue.
GET /api/v1/placements JWT
List placements with optional status filter. Paginated (limit/offset).
GET /api/v1/timesheets JWT
List timesheets with optional week_ending and status filters. Ordered by week descending.
GET /api/v1/vms/uploads JWT
Last 50 VMS file uploads for the tenant.
POST /api/v1/vms/uploads/presign JWT
Generate pre-signed S3 URL for VMS file upload. Creates upload tracking record.
GET /api/v1/agents/runs JWT
Agent run history with optional agent_type filter.
GET /api/v1/agents/runs/{run_id}/results JWT
Detailed results for a specific agent run. Filter by severity and review status.
POST /api/v1/agents/results/{result_id}/review JWT
Submit review decision: approved, escalated, corrected, or dismissed. Records reviewer and timestamp.
GET /api/v1/audit JWT
Audit trail for the tenant. Every agent action, review decision, and data change.

Intelligence Layer

Three-Tier Memory Architecture

Agents get smarter over time. Client-specific learned patterns, tolerance overrides, and known exceptions persist across runs so the system adapts to each customer's operations.

⚡
Tier 1

Transaction Memory

LangGraph state per agent run. Pydantic-validated. Includes messages, results, review decisions, token usage. Ephemeral — lives for the duration of one task. Audit log persists the record.

📚
Tier 2

Client Memory

Per-tenant persistent configuration: tolerance overrides (e.g., "GHR allows 5% OT variance"), known exceptions, VMS quirks, billing rules, escalation contacts, and learned patterns from past runs.

🌎
Tier 3

Platform Memory

Cross-tenant intelligence (planned for 8+ customers). Common VMS parsing patterns, industry benchmarks, regulatory updates. Anonymized — no tenant data crosses boundaries.

Security & Compliance

Enterprise Security by Design

Built for staffing firms handling sensitive PII, financial data, and regulatory compliance requirements across multiple jurisdictions.

🔒

Tenant Isolation

PostgreSQL Row-Level Security on every data table. Impossible to query another tenant's data even with direct DB access.

🔐

Encryption

TLS in transit (ACM certificates). AES-256 at rest (Aurora, S3). Secrets in AWS Secrets Manager — never in code or env files.

👥

Authentication

JWT with HS256 signing. 8-hour token expiry. bcrypt password hashing. Role-based access: admin, manager, viewer.

📜

Audit Trail

Every agent action logged with SHA-256 input hash for tamper detection. Reviewer identity, timestamp, and decision recorded.

🛠

Human-in-the-Loop

No autonomous writes to production data. Approval gates for financial modifications, client communications, and compliance overrides.

🌐

Network Isolation

VPC with private subnets for compute and database. NAT gateway for outbound. No public IP on any service except ALB.

🚩

No Hiring AI

Deliberate boundary: no candidate screening or hiring agents. Avoids EEOC/OFCCP bias risk and EU AI Act high-risk classification.

🕵

PII Protection

GDPR-simulated privacy standards. PII redacted from debug/test output. No candidate data in LLM prompts without tenant consent flags.
