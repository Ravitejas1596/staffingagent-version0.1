# StaffingAgent.ai Product Guide

This guide explains what the product does, who uses it, and how each part of the Command Center helps daily staffing operations.

## 1) What problem this product solves

Staffing companies lose money and time when ATS data, timesheets, VMS records, payroll, and invoices do not align.

Typical problems:
- hours are missing or duplicated
- bill rates and pay rates are incorrect
- invoice values do not match source data
- risk and compliance issues are found too late

StaffingAgent.ai continuously checks these gaps and gives teams clear, actionable recommendations.

## 2) Who uses it

- **Operations leaders** monitor performance and risk.
- **Recruiters and analysts** review and approve actions.
- **Admins** manage users, settings, and client integrations.
- **Super admins** manage multi-tenant clients and platform-level controls.

## 3) How users work in the product

Daily routine:
1. Log in with email, password, and tenant slug.
2. Open Command Center to see current status.
3. Review pending actions from agents.
4. Approve or reject recommended actions.
5. Execute approved actions and track results.

## 4) Command Center explained component by component

### Summary Row
Shows immediate health indicators:
- active workload
- pending actions
- active agent activity
- current impact trends

### Action Queue
The most important operational panel.

What it does:
- lists items that need human review
- shows what the agent recommends and why
- lets users open specific workflows to approve/reject

### Agent Fleet Panel
Shows each available agent and status.

What users can do:
- see whether agent is idle/running/error
- manually run an agent plan
- navigate directly into that agent’s detailed screen

### ROI Panel
Summarizes business impact in financial terms (recovered value, trend, efficiency indicators).

### Quality Metrics
Displays quality scores and health indicators so teams can catch drift before it becomes costly.

### Charts and Activity Feed
- trend charts for processing/throughput and impact over time
- utilization chart to show where agent effort is concentrated
- live activity feed to show what happened recently

## 5) Main product screens

- **Command Center**: central mission-control view
- **Dashboard**: deeper operational panels with filters and run control
- **Alert Queue**: dedicated alert lifecycle management
- **TimeOps / RiskOps**: focused operations views
- **Agent Plan View**: plan, approve, execute lifecycle for most agents
- **User Management**: user roles, invites, access operations
- **Client Management**: tenant-level controls for super admins
- **Agent Settings**: per-agent tuning and message template controls

- VMS Matching

- Payroll Reconciliation

### Common behavior across agents
1. Pull tenant-scoped data.
2. Detect potential issues.
3. Create a review plan with action items.
4. Wait for human approval where needed.
5. Execute approved actions.
6. Store results and audit trail.

## 7) Human-in-the-loop model

This product is built for controlled automation, not blind automation.

- Important actions can require approval first.
- Users can reject suggestions.
- Alert event history is append-only for accountability.
- Reversal workflows exist for safety in key alert operations.

## 8) Data and trust model (non-technical summary)

- Your organization’s data is isolated from other clients.
- Every action is tied to a user/agent and timestamp.
- Sensitive integration credentials are protected.
- The system keeps detailed logs for governance and audit readiness.

## 9) What success looks like

Teams using this correctly should see:
- faster issue detection
- shorter resolution times
- fewer billing and payroll errors
- improved operational confidence
- measurable financial recovery from leakage prevention

## 10) Quick start checklist

1. Confirm integrations and baseline data sync.
2. Validate user roles and permissions.
3. Start with Command Center and Action Queue daily.
4. Run core agents on a regular cadence.
5. Review weekly impact trends and adjust settings where needed.
